# USB updater

Небольшой AutoIt-скрипт для пакетного обновления USB-накопителей, подключённых к хабу.

После подключения новых накопителей он копирует на каждый содержимое папки `data`,
перезаписывая совпадающие файлы, затем безопасно отключает том. Когда обработана вся
партия, появляется уведомление, что хаб можно менять.

Готовую программу можно скачать на [странице релизов проекта](https://github.com/insan3d/usb-updater/releases).

## Подготовка

1. Положите нужные файлы и папки в папку `data` рядом с программой `.exe`.
   При первом запуске программа создаёт эту папку автоматически.
2. Не размещайте в `data` саму папку назначения или подключённые накопители.

## Как пользоваться

1. Запустите программу.
2. Подключите все нужные накопители к USB-хабу.
3. Включите чекбокс отслеживания.
4. Подключите хаб к компьютеру одним действием.
5. Подождите: после последнего системного события скрипт ждёт 2.5 секунды, чтобы собрать
   всю партию, затем копирует данные.
6. Дождитесь уведомления **«Хаб можно менять»**.

В чёрном окне отображается журнал действий. После успешного отключения буква тома
исчезает из Проводника; накопитель можно отсоединять от хаба.

## Важно

- Накопители должны появиться в системе **после** включения чекбокса. Поэтому сначала
  заполните хаб, включите отслеживание, а затем подключите хаб к компьютеру. Это защита
  от случайной перезаписи уже подключённых дисков.
- Содержимое `data` копируется рекурсивно, существующие файлы на накопителе
  перезаписываются. Лишние файлы с накопителя не удаляются.
- Некоторые флешки Windows определяет как `FIXED`, поэтому скрипт реагирует на любой
  новый локальный съёмный или фиксированный том. Не подключайте в это время сторонние
  внешние диски, виртуальные диски или другие тома, которые не нужно обновлять.
- При сообщении об ошибке не отключайте накопитель: сначала прочитайте последнюю
  строку журнала и устраните причину (защита от записи, нехватка места и т. п.).
